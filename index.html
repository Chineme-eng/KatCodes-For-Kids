👅 LANGUAGES

What language should you learn first?

Every programming language is a tool. Here's what the most popular ones look like — all doing the same thing: printing "Hello, World!"

🌐 HTML — The skeleton of every website. It gives pages structure.

<!-- This is an HTML comment -->
<h1>Hello, World!</h1>
<p>I'm learning to code!</p>
<button>Click me</button>

🏁 Best for: Anyone who wants to build websites. Start here if you're not sure!

🎨 CSS — The paint and decorator. It makes things look beautiful.

/* Change the page background color */
body {
  background-color: coral;
  font-family: sans-serif;
  color: white;
}

h1 {
  font-size: 48px;
}

🏁 Best for: People who care about design and making things look great.

⚡ JavaScript — The brain. Makes websites interactive and alive.

// Show a greeting
let name = "World";
console.log(`Hello, ${name}!`);

// React to a button click
document.querySelector("button")
  .addEventListener("click", () => {
    alert("You clicked me! 🎉");
  });

🏁 Best for: Web development, games, apps — incredibly versatile!

🐍 Python — Super readable. Great for beginners and professionals alike.

# Print a greeting
print("Hello, World!")

# A simple function
def greet(name):
    return f"Hello, {name}! 👋"

print(greet("World"))

🏁 Best for: Data science, AI, automation, beginners — reads almost like English!

🍎 Swift — Apple's language for iPhone and Mac apps.

// Swift - used for iOS apps
import Foundation

let greeting: String = "Hello, World!"
print(greeting)

func greet(name: String) -> String {
    return "Hello, \(name)! 🍎"
}

🏁 Best for: Building iPhone or Mac apps. You'll need a Mac to get started.

🗺️ ROADMAP

Your path from zero to hero.

Don't try to learn everything at once. Follow this order and you'll build confidence every step of the way.

1

🌐 HTML Basics (Week 1–2)

Learn what tags are, how pages are structured, and build your first webpage. Takes about 5–10 hours total.

2

🎨 CSS Fundamentals (Week 2–4)

Make your page look good! Colors, fonts, spacing, flexbox for layout. This is where things start feeling real.

3

⚡ JavaScript Basics (Month 2)

Add interactivity — buttons, forms, animations. This is the "magic" language that makes sites actually work.

4

🐙 Git & GitHub (Month 2–3)

Save your work, undo mistakes, and share your projects. GitHub Pages lets you host real websites — free!

5

🚀 Build Projects! (Month 3+)

This is where learning explodes. Build things you care about. Show them off. Get feedback. Repeat!

🌙 HOW THIS PAGE WORKS

How the dark/light mode on this page works.

Want to add dark mode to your own site? Here's exactly how this page does it — it's simpler than you think!

1

CSS Variables for colors

Instead of writing color: orange everywhere, we use variables like --accent: #ff8c00 in :root {}. Then for dark mode, we just redefine those same variables.

2

A data attribute on <html>

The HTML element gets data-theme="light" or data-theme="dark". CSS uses [data-theme="dark"] { } to override the colors.

3

One tiny JavaScript function

A single function toggles the attribute: document.documentElement.dataset.theme = "dark". That's it! Smooth transitions are handled by CSS.

4

Remember the preference

We save the user's choice with localStorage.setItem("theme", "dark") so it persists on page reload.

/* Step 1: Define colors as variables */
:root {
  --bg: #fffaf3;
  --text: #2d2013;
  --accent: #ff8c00;
}

/* Step 2: Override for dark mode */
[data-theme="dark"] {
  --bg: #16131f;
  --text: #f0eaff;
  --accent: #f7c948;
}

/* Step 3: Use variables everywhere */
body { background: var(--bg); color: var(--text); }

/* Step 4: Smooth transitions */
body { transition: background 0.35s, color 0.35s; }
// JavaScript toggle function
function toggleTheme() {
  const html = document.documentElement;
  const isDark = html.dataset.theme === "dark";
  html.dataset.theme = isDark ? "light" : "dark";
  localStorage.setItem("theme", html.dataset.theme);
}

🐙 GITHUB PAGES

How to publish this page (and your own!) for free.

GitHub Pages hosts static websites completely free. Here's the step-by-step from scratch:

1

Create a GitHub account

Go to github.com → click "Sign up" → it's free. Use a professional username — it'll be in your URL!

2

Create a new repository

Click the + icon → "New repository" → name it my-coding-guide → check "Add a README" → click "Create repository".

3

Upload your index.html

In your new repo, click "Add file" → "Upload files" → drag your index.html file in → scroll down → click "Commit changes".

4

Enable GitHub Pages

Go to your repo → click "Settings" → scroll to "Pages" in the left sidebar → under "Branch" select main → click "Save".

5

🎉 Your site is live!

Within ~60 seconds, your site will be at https://yourusername.github.io/my-coding-guide. Share it with everyone!
